切成 token 之后，tokenizer 就靠这张有限的词表，把每个 token 查成一个固定的 id —— 能编码，也能解码
那这张词表怎么设计？两个极端：整词太大，单字太碎
维度 Word-based一个词一个 token Character-based一个字符一个 token
切分示例
单 token 语义 ✓丰富、完整 ✕几乎没有
序列长度 ✓短，attention 算得快 ✕极长，计算量随长度平方涨
未知词 OOV ✕遇到新词就 [UNK]，信息丢失 ✓100% 覆盖，几乎没有 [UNK]
词表大小 ✕极大 · 几十万 ~ 上百万 ✓极小 · 几十 ~ 几千
